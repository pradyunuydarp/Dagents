## Live Kubernetes Validation With Minikube

- Status: pending
- Goal: validate that Dagents Kubernetes manifests generated by the core service can be applied to a live Minikube cluster on macOS, and confirm the deployed agents and services start correctly and answer real health/API calls.

### Scope

- Start a local Minikube cluster on macOS.
- Build or load the required Dagents images into the Minikube runtime.
- Generate manifests from the live core-service API.
- Apply the generated manifests to Minikube.
- Verify Pods, Services, and any generated ConfigMaps/ServiceAccounts are created as expected.
- Confirm `lma`, `gma`, `core-service`, `model-service`, and `pipeline-service` become healthy in-cluster.
- Exercise live API checks against the deployed workloads using `kubectl port-forward` or Minikube service access.

### Prerequisites

- Docker Desktop installed and running on macOS.
- Minikube installed locally.
- `kubectl` installed locally and pointed at the Minikube context.
- Sufficient Docker Desktop disk and memory for Python and Spring image builds.
- Local env files available from `env/`.

### Execution Notes

- Prefer a dedicated Minikube profile for Dagents validation.
- Use `minikube start` with explicit CPU and memory settings so image builds and Pods do not thrash.
- Decide whether to:
  - build images directly inside Minikube with `eval $(minikube docker-env)`, or
  - build locally and `minikube image load` each required image.
- Confirm image tags used in generated manifests match the images loaded into Minikube.
- Generate manifests from the actual running core-service endpoint instead of hand-writing YAML.
- Save the generated YAML used for the run so the exact deployment payload can be audited afterward.
- Apply manifests in a dedicated namespace such as `dagents-dev`.
- Watch rollout status with `kubectl get pods -n dagents-dev -w` and inspect failures with `kubectl describe` and `kubectl logs`.
- Validate service reachability with `kubectl port-forward` before relying on broader network exposure.

### Validation Checklist

- Minikube cluster starts successfully.
- Namespace is created and active.
- All required images are available to the cluster.
- Manifest generation request succeeds from `core-service`.
- Generated manifests apply cleanly with `kubectl apply`.
- All expected Kubernetes resources are present.
- All deployed Pods reach `Running` and `Ready`.
- Health endpoints respond successfully for each deployed service.
- LMA and GMA control-plane calls work from the deployed environment.
- Any manifest generation defects or runtime mismatches are captured for follow-up fixes.

### When We Start This Task

- Re-check local Docker Desktop capacity before building.
- Bring up the repo services needed to generate manifests.
- Start Minikube and confirm `kubectl config current-context`.
- Choose the exact image-loading strategy for macOS.
- Run one narrow deployment first, then expand to the full Dagents set once the first manifest path is proven.
